Fix property listing error with comprehensive debugging

This commit adds extensive error logging and fixes to resolve the
"Failed to create property" error on the deployed application.

Backend Changes:
- Fix property number generation to use mongoose.model() for reliability
- Add error handling in property number uniqueness check
- Add comprehensive logging before database save
- Handle empty/invalid rooms data
- Improve Cloudinary upload error handling
- Log full payload and validation errors

Frontend Changes:
- Add form data logging before submission
- Log backend response for debugging
- Show detailed field-level error messages
- Extend error toast duration to 5 seconds

Files Modified:
- backend/src/tables/property.js
- backend/src/routes/properties.js
- frontend/src/pages/UploadProperty.jsx

Documentation Added:
- PROPERTY_LISTING_FIX.md - Fix overview and deployment guide
- DEBUGGING_GUIDE.md - Step-by-step debugging instructions

These changes will help identify the exact cause of the property
creation failure through detailed logging in both browser console
and Render backend logs.
